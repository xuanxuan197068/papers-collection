## A. 中立全景

### 总体判断

NDSS 2025 最明显的技术变化不是单纯“用了更多 AI”，而是安全研究开始把 AI 与外部反馈、系统状态、硬件边界和物理世界结合起来：LLM 生成的规则需要执行或证明器复核；模糊测试需要恢复真实状态或学习行为模型；检测系统把规则、梯度、溯源图与因果干预结合；隔离从进程边界下沉到内核分区、数据流和加速器 I/O；物理攻击则从单一信号注入扩展到跨模态耦合。

### 1. ML 安全：从分类器攻防走向输入重生成、潜空间与机器遗忘

- **Diffence（6692）**：在推理前用扩散模型重生成输入，抹平成员与非成员的差异 → 解决成员推断防御通常需要改模型或牺牲效用的问题 → 用于敏感图像分类模型在线推理 → **benchmark**。
- **CENSOR（6686）**：在原梯度的正交子空间做贝叶斯采样，选择兼顾效用与隐私的替代梯度 → 阻止联邦学习梯度被反演为训练样本 → 用于高维大型神经网络的客户端—服务器训练 → **benchmark**。
- **Scale-MIA（6846）**：先闭式反演潜空间，再用生成解码器批量恢复输入 → 绕过安全聚合并降低逐批优化成本 → 用于恶意参数服务器下的联邦学习 → **benchmark**。
- **Reinforcement Unlearning（6720）/ TrajDeleter（6738）**：分别删除整个训练环境或指定离线轨迹的影响 → 解决 RL Agent 接到数据撤回、污染清除请求后无法低成本重训的问题 → 用于在线/离线强化学习 → **unknown / benchmark**。

**技术倾向**：隐私攻防正在从输出置信度转向输入、梯度和潜空间；“删除模型已学知识”也从监督学习扩展到环境和轨迹级 RL 遗忘。

### 2. LLM：既成为安全工具，也成为需要系统隔离的新执行体

- **GPTAid（6698）**：LLM 从源码生成 API 参数安全规则和正反调用代码，再用动态执行反馈过滤、用代码差分细化 → 解决纯 LLM 规则幻觉、不可执行和误报高的问题 → 用于程序库 API 误用检测 → **benchmark**。
- **PropertyGPT（6830）**：检索人写属性供 LLM 参考，用编译/静态分析反馈迭代修正，最后交给证明器验证 → 解决智能合约形式验证依赖专家手写属性的问题 → 用于合约属性生成和漏洞证明 → **benchmark**。
- **YuraScanner（6884）**：让 LLM 作为目标驱动 Agent 理解网页语义并执行完整工作流，再对到达的输入点做 XSS 测试 → 解决扫描器不理解业务流程、无法进入深层状态的问题 → 用于现代 Web 应用自动扫描 → **benchmark**。
- **IsolateGPT（6807）**：给 LLM、第三方应用及自然语言交互建立隔离边界和受控通信 → 解决 Agent 应用自由访问数据和相互调用带来的权限扩散 → 用于插件化 LLM 系统 → **benchmark**。
- **The Philosopher’s Stone（6732）/ PROMPTPEEK（6801）**：前者给第三方低秩适配器植入触发式木马，后者利用跨租户 KV-cache 共享恢复提示词 → 分别揭示 LLM 供应链和推理基础设施攻击面 → **benchmark**。

**技术倾向**：有效的 LLM 安全系统普遍不再信任一次生成结果，而是引入执行、编译、静态分析或形式证明作为外部 oracle；与此同时，LLM Agent 已需要类似传统操作系统的隔离、权限和供应链模型。

### 3. 自动化测试：状态、结构和语义成为 Fuzz 的一等输入

- **Moneta（6710）**：快照重托管 + 记录重放 + GPU 栈虚拟化，恢复真实运行状态后并行离体 fuzz → 解决 GPU 驱动依赖物理硬件、深层状态和时序难复现的问题 → NVIDIA/AMD/Mali 驱动 → **benchmark**。
- **Truman（6870）**：从开源 OS 驱动推断消息及状态依赖，构建设备行为模型指导闭源虚拟设备 fuzz → 解决随机输入无法满足设备协议依赖的问题 → QEMU、VirtualBox、VMware、Parallels → **real-world**。
- **ICSQuartz（6803）**：针对 PLC 扫描周期设计原生变异策略 → 解决通用 fuzzer 忽略工控程序周期语义和厂商封闭性的问题 → IEC 61131-3 Structured Text 与真实 ICS 库 → **benchmark**。
- **EvoCrawl（6696）/ YuraScanner（6884）**：分别用进化搜索和 LLM 目标 Agent 生成 Web 交互序列 → 解决表单约束、业务工作流和服务器状态阻碍漏洞路径触达的问题 → **benchmark**。
- **DUMPLING（6786）/ TWINFUZZ（6872）**：通过优化前后帧状态或软硬件解码结果做细粒度差分判定 → 解决仅靠崩溃信号无法发现静默错误的问题 → JavaScript JIT 与视频硬件栈 → **real-world**。

**技术倾向**：Fuzz 的关键创新从“怎么变异字节”转为“怎样构造正确的状态、依赖和工作流”，判定器也从 crash 扩展为状态差分、行为差分和语义目标达成。

### 4. 系统隔离：硬件原语与数据流驱动的细粒度边界

- **BULKHEAD（6682）**：Intel PKS + 轻量内核监控器 + 两级分区 → 在 Linux 内核模块之间提供双向隔离，同时维持扩展性和低开销 → **benchmark**。
- **ASGARD（6757）**：虚拟化 TEE、安全 I/O 直通、TCB 裁剪和退出合并 → 把片上 NPU 纳入可信边界且兼容闭源驱动 → 用于 Arm Android 端侧 DNN 推理 → **real-world**。
- **RContainer（6835）**：扩展 ARM CCA，以可信 mini-OS 和逐容器物理地址空间隔离不可信 OS → 用于云平台强隔离容器 → **real-world**。
- **TZ-DATASHIELD（6873）**：LLVM 按敏感数据流自动划分 TrustZone 隔间，并在运行时验证资源访问 → 降低 MCU 手工隔离成本并保护共享敏感数据 → 医疗、工业自动化和无人载具固件 → **real-world**。

**技术倾向**：隔离粒度从 VM/进程继续下沉到内核组件、容器地址空间、TEE 内部和敏感数据流；评价重点同时覆盖 TCB 大小、设备 I/O、自动化划分和真实硬件开销。

### 5. 物理与侧信道：攻击开始跨越信号模态和系统层级

- **PhantomLiDAR（6715）**：用有意电磁干扰耦合激光雷达接收、监测和波束转向模块，注入/删除点云甚至关闭设备 → 解决传统防护只考虑激光注入的问题 → 自动驾驶商用 LiDAR → **real-world**。
- **PowerRadio（6716）**：经建筑接地线注入信号并耦合到模拟测量线，绕过滤波器篡改传感读数 → 解决封装和软件防护无法阻止远距离供电网络注入的问题 → 摄像头、麦克风和环境传感器 → **real-world**。
- **ReThink（6721）**：向逆变器传感器发射高频电磁干扰，欺骗控制算法造成停机、损坏或降功率 → 揭示通过 EMC 测试的设备仍存在安全耦合 → 光伏逆变器与真实微电网 → **real-world**。
- **Off-Path TCP Hijacking（6712）/ PROMPTPEEK（6801）**：从加密 Wi-Fi 包尺寸或共享 KV-cache 资源中恢复逻辑状态 → 表明侧信道正从硬件缓存扩展到协议和 AI 服务资源共享 → **real-world / benchmark**。

**技术倾向**：研究对象从“数字系统泄漏秘密”扩展到“物理耦合改变控制决策”；有效防御需要同时理解电路、协议、共享资源和上层控制算法。

### 6. 检测与诊断：规则、学习、溯源和因果的混合化

- **CAPTAIN（6701）**：把规则标签传播变成可微过程，用梯度下降优化节点、边和告警阈值 → 让轻量可解释 PIDS 自动适配不同环境并减少误报 → DARPA 与模拟主机环境 → **benchmark**。
- **ProvGuard（6832）**：静态分析指导 SDN 控制器插桩，构建溯源图并用路径上下文和序列预测误差检测策略操纵 → 解决只看数据平面无法识别控制决策因果链的问题 → Floodlight SDN → **benchmark**。
- **IRCA（6805）**：基于多传感器融合 DAG 建分层结构因果模型，以干预和分治剪枝定位根因 → 解决多模块故障耦合时异常检测只能告警、不能解释根因的问题 → Autoware/Apollo 自动驾驶感知 → **benchmark**。
- **ERW-Radar（6695）**：结合 I/O 重复行为上下文、字节流内容和自适应监控策略 → 识别刻意模仿良性程序的规避型勒索软件 → 终端实时防护 → **benchmark**。

**技术倾向**：纯黑盒分类器与纯静态规则之间出现中间路线——保留规则、图和因果结构的可解释性，再用梯度或序列模型做自适应。

### 7. 密码与区块链：从新原语转向可部署协议和实现验证

- **PQConnect（6827）**：自动发现对等方，以分层后量子 KEM 建立应用无关的端到端 IP 隧道 → 解决存量应用难以无改造迁移到后量子保护的问题 → 客户端到任意支持服务器 → **theory**。
- **MTZK（6711）**：用变形关系自动变异零知识 DSL，并比较原始与变体编译结果 → 发现 ZK 编译器错误编译导致无效证明被接受的问题 → 四类工业 ZK 编译器 → **benchmark**。
- **Mysticeti（6817）**：无显式认证 DAG + 新提交规则，把资产快速路径编织进共识图 → 降低 DAG 认证和提交等待造成的延迟 → 广域网区块链共识 → **real-world**。
- **Siniel（6854）**：秘密共享私密见证并完全外包 zkSNARK 证明计算 → 解决资源受限证明者高开销和持续参与 MPC 的问题 → 私密支付、合约和匿名凭证 → **benchmark**。

**技术倾向**：热点由“构造新密码原语”逐步转向迁移、编译器正确性、证明外包、跨片/跨链以及真实网络性能。

### 8. 网络与基础设施：大规模测量和协议状态推理仍是 NDSS 强项

- **ReDAN（6719）**：利用 NAT 的 PMTUD 规范缺陷识别设备并删除无辜 TCP 映射 → 解决网外攻击者如何远程中断 NAT 内部连接的问题 → 家用、移动、公共 Wi-Fi 与云网络 → **real-world**。
- **SKYFALL（6736）**：刻画 LEO 时变瓶颈并由受控终端持续拥塞链路 → 解决传统链路洪泛防护无法适应卫星动态拓扑的问题 → 运行中的低轨卫星网络 → **simulation**。
- **ASPAwN（6850）**：在 ROV 普及后的威胁模型下扩展 ASPA 路径验证 → 解决新型 BGP spoofing 和部分部署条件下的防御缺口 → 互联网路由系统 → **simulation**。
- **PROXY Protocol 测量（6752）**：全 IPv4 扫描并注入伪造 PROXY 头 → 发现错误信任配置可伪造客户端、绕过代理并泄露内网 → HTTP、IoT、智能家居和 SMTP → **real-world**。

### 9. 人因与治理：安全机制是否被理解成为独立研究问题

- **The Power of Words（6733）**：把权限理由措辞和界面元素做组合实验 → 判断什么表达会改变用户授权决策 → Google Play 权限请求 → **user study**。
- **iOS App Privacy Report（6739）**：访谈用户对数据、传感器和网络活动披露的理解与补救选择 → 揭示透明度可能变成信息过载 → iOS 隐私报告 → **user study**。
- **Where Are We On Cyber?（6749）**：访谈董事、CISO 和顾问，分析信息依赖、权力失衡与预算化 → 解释董事会网络风险监督为何失效 → 英国大型企业治理 → **user study**。
- **Ctrl+Alt+Deceive（6774）**：用大规模终端遥测比较七类诈骗的真实曝光、地域和广告引流 → 解决诈骗研究只看域名数量、不了解用户暴露的问题 → 桌面与移动终端 → **real-world**。

## B. 结合研究方向的迁移观察

### 直接命中

- **自主网络防御（9 篇）**：ERW-Radar、CAPTAIN、MineShark、Density Boost、分阶段勒索检测、EAGLEYE、Hitchhiking Vaccine、VulShield、YuraScanner。共同信号是从“离线判别”转向在线自适应、主动探测、自动修复和任务执行。
- **RL / 强化学习（2 篇）**：Reinforcement Unlearning 和 TrajDeleter 都聚焦删除训练影响。它们不直接研究攻防决策，但非常适合自主防御 Agent 的污染恢复、策略合规和任务轨迹撤回。
- **蜜罐与网络欺骗（3 篇）**：数据库勒索研究用蜜罐观测攻击，HADES 展示蜜罐型邮件黑名单基础设施可被反向利用，设备搜索引擎研究用 28 个蜜罐长期测量扫描策略。这里的价值更多是“欺骗传感器的测量与抗操纵”，而非新型交互式蜜罐。
- **未发现严格直接命中**：world model、多智能体、靶场生成、开集识别。以下迁移线索因此是机制层推断，不是论文原始主张。

### 跨域迁移线索

1. **自主靶场探索器**：把 YuraScanner 的目标驱动 Agent、EvoCrawl 的进化式状态探索、Truman 的设备行为模型组合起来，可形成“理解任务 → 生成合法动作序列 → 保持状态依赖 → 自动验证漏洞”的靶场红队 Agent。这里最可迁移的是状态建模和动作约束，不是具体 Web/设备实现。
2. **可解释的自主防御闭环**：CAPTAIN 提供可微规则参数，ProvGuard 提供溯源路径上下文，IRCA 提供因果干预定位。三者可拼成“检测 → 解释 → 定位 → 调参”的闭环，适合自主防御系统避免纯黑盒策略。
3. **Agent 的遗忘与污染恢复**：Reinforcement Unlearning 删除环境影响，TrajDeleter 删除具体轨迹。可迁移到攻击者诱导、错误演练场景或过时战术污染后的策略回滚，并为 world model/策略模型设计可审计遗忘。
4. **蜜罐不只是诱饵，也是可被攻击的传感器**：HADES 表明攻击者可以操纵依赖诱捕服务器的检测基础设施；设备搜索引擎研究又表明蜜罐能长期刻画扫描者策略。下一步值得研究“抗投毒蜜罐群”和“多视角一致性验证”，防止欺骗系统反被欺骗。
5. **物理攻击可转化为靶场场景生成器**：PhantomLiDAR、PowerRadio、ReThink 和 IRCA 给出了“物理注入 → 多模块故障 → 因果定位”的完整链条，可用于生成 cyber-physical range 的故障与对抗场景，训练自主防御 Agent 处理非纯网络攻击。
6. **LLM Agent 需要传统系统安全护栏**：IsolateGPT 的执行隔离、插件木马研究的供应链威胁、PROMPTPEEK 的共享缓存泄漏共同说明，多智能体攻防平台需要隔离域、能力令牌、调用审计和租户级资源清理，而不能只靠提示词约束。
