# 欺骗式反制自主攻击 agent（蜜标+陷阱利用 LLM 架构弱点）

- **论文**: Cloak, Honey, Trap: Proactive Defenses Against LLM Agents
- **venue**: USENIX Sec 2025　**链接**: https://www.usenix.org/conference/usenixsecurity25/presentation/ayzenshteyn
- **评分**: N=4 R=5 T=5
- **成卡日期**: 2026-07-11

## 机制（领域无关描述）

把"欺骗防御"的作用对象从人类攻击者/自动扫描器换成**由 LLM 驱动的自主攻击体**，
攻击面锁定 LLM 本身的架构性弱点而非其提示内容：认知偏置、上下文/记忆容量上限、
分词边界。基于这些弱点造三类干预件——(1) cloak：用误导信息把真实资产藏进噪声，
利用 agent 注意力/上下文预算有限；(2) honey-token：投放对 LLM 特别"诱人"的假凭据/线索，
命中即暴露；(3) trap：用循环、自指、超长上下文等把 agent 拖死或诱其执行不可信代码，
反向获得对攻击者基础设施的访问。关键点：6 策略 15 技术里多数**不依赖 prompt injection**，
而是结构性地攻击"自主决策循环"这一通用弱点。黑盒假设即可生效。

## 原始场景

自动化渗透测试 agent（LLM 驱动、能自主攻陷主机）被反用为攻击工具的场景。在 11 台
CTF 靶机上部署，100% 成功保护。配套开源工具 CHeaT 自动把陷阱/伪装/蜜标注入网络资产。

## 解决的问题

LLM 让攻击可规模化自动化，传统防御（补丁、检测签名）针对人类节奏设计，跟不上自主
agent 的速度与规模；且攻方与防方用同一套 AI 能力，防守方缺少专门针对"自主体"的杠杆。

## 结果与证据

11 台不同 CTF 机器 100% 保护成功率（真实工具级验证，非纯仿真）；演示了诱导 agent 执行
不可信代码从而反向进入攻击者设施的新型 exploit。证据强度：中上（CTF 环境真实但受控，
未及生产网络长期对抗）。

## 迁移假设

直接落到"自主网络防御 + 蜜罐/欺骗 + 多智能体"交叉点：
- **输入→输出映射**：把防御方的资产伪装/蜜标布设建模成"针对攻击 agent 决策循环的对抗
  扰动生成"问题，输入=攻击 agent 的可观测状态，输出=最大化其决策偏离的欺骗配置。
- **预期收益**：给我的方向提供一个"防御方主动利用攻方 AI 弱点"的范式，而非被动检测。
  可与 RL 结合——把欺骗件的投放策略做成 RL 策略（状态=攻方行为轨迹，动作=布设哪种
  cloak/honey/trap，奖励=攻方偏离/暴露程度），CAMP（见 [[camp-provably-robust-rl]]）
  的认证鲁棒可叠加，保证欺骗策略在攻方自适应下不崩。
- **最可能失效处**：攻方换非 LLM 或带反蜜罐检测的 agent 后弱点假设失效；黑盒假设在
  攻方能大量试探时被侵蚀。需要"欺骗有效性"随攻方能力演化的动态模型（呼应 world model）。

## 备注/风险

复现门槛低（开源 CHeaT + CTF 环境）。与 [[coordinated-iot-honeypots]] 互补：那篇解决
"多蜜罐跨层一致性"（骗核验），这篇解决"骗认知"，二者可组合成"认知+物理双层欺骗"。
